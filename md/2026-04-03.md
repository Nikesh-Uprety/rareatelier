### 2.3 Guest Cart Persistence and Reload Recovery
Files:
- `client/src/store/cart.ts`
- `client/src/pages/storefront/ProductDetail.tsx`
- `client/src/pages/storefront/Cart.tsx`
- `client/src/pages/storefront/Checkout.tsx`
- `client/src/components/layout/CartSidebar.tsx`
- `server/routes.ts`
- `server/services/cartService.ts`
- `tests/unit/cart-store.test.ts`

What changed:
- Added stable guest cart identity via `guestId`.
- Added Redis-backed guest cart sync/read APIs:
  - `POST /api/cart/guest/sync`
  - `GET /api/cart/guest/:guestId`
- Added local browser backup plus server rehydration flow.
- Added cookie fallback so guest identity can be restored if `localStorage` is missing.
- Forced guest cart fetches to bypass stale cached `304` behavior during reload hydration.

Manual QA:
- Add a product to bag, hard reload, confirm bag items remain.
- Close tab, reopen site, confirm bag items remain.
- Add multiple products from different product pages and verify all restore after reload.
- Verify cart sidebar, full cart page, and checkout all show the same restored items.
- Check browser devtools:
  - guest cart sync requests return `200`
  - guest cart fetch returns `200`
  - guest cookie is present

Known limitation:
- If both `localStorage` and the guest cookie are manually cleared, the previous Redis cart becomes unreachable because the guest identifier is lost.
